# Test Plan: Personal Korean Stock Auto-Trading Dashboard

Generated by /plan-eng-review on 2026-04-03

## Affected Pages/Routes
- `/` — Main dashboard (portfolio, chart, signals, trade log)
- `/api/ws` — WebSocket endpoint for real-time data streaming
- Backend services: broker_adapter, strategy, orders, ws hub

## Key Interactions to Verify
- Real-time price updates appear on dashboard via WebSocket
- Strategy signals appear with Execute/Ignore buttons
- Order placement via Execute button triggers broker API and updates trade log
- Emergency stop cancels all pending orders and halts strategies
- Auto/manual mode toggle changes signal handling behavior
- Portfolio panel refreshes on polling interval and after order execution

## Edge Cases
- KIS API credentials missing or invalid at startup
- WebSocket disconnect/reconnect (staleness indicator shows)
- Strategy script missing `run()` function or returning invalid signal
- Order exceeds position size limit (risk control rejects)
- Daily loss limit reached (all new BUY signals rejected)
- Max 40 WebSocket subscriptions reached
- Partial order fill (logged but not re-ordered)
- Strategy script blocks for >30s (timeout)
- App started outside market hours (scheduler idles)

## Critical Paths
- Morning startup: app → KIS auth → portfolio load → ticker subscribe → dashboard ready
- Signal → Order: strategy fires signal → risk validation → order placement → broker confirmation → trade log + portfolio update
- Emergency stop: button click → cancel pending orders → halt engine → broadcast HALTED → UI shows stopped state
